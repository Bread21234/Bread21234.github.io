My Past Isp

Concept:

The goal of my team's isp was to create a 2D bullet-hell rogue like. In the game, the player would traverse a map of randomly generated rooms, fighting enemies and obtaining items along the way. We themed our game around the underworld and the player's decent into hell.

Motivation:

Most of the motivation came from my group's love of the genre. I spent many hours in a game called "Enter the Gungeon" and it inspired me to build this isp. Additionally, creating a videogame has always been a dream of mine, so I was excited to get a chance.

Code Segments and Explanations:

Firstly, when the player enters the room, their 2D box collider interacts with the room. Then, the game checks whether or not that room has already been completed. If the room has not been completed, the room is locked by instantiating locked doors on all of the openings. The game then generates random enemies and a number of enemies based on what floor the player is on.

On lines 7-14, I define GameObjects and integers for later. Immediately as the game begins, the retry button is hidden, the health is set to its max, and the actual healthbar displays the new info. The game checks constantly for whether or not health changes. If the player takes damage and health decreases, the current health is adjusted and along with it, the healthbar is too. If at any point the player loses all of their life, the death effect is plated and the retry button is revealed to the user.

Conclusions:

One of the most crucial things I learned during my isp is that learning a new skill is much easier when there is a large, supportive community to help guide you. The project was possibly the most interesting project I had ever completed, and it was very satisfying to finish. I expected about the amount of code I needed, but I did not expect how much I needed to break things down. I was also pleasantly surpised with how intuitive both Unity and C# were; both were very user friendly and provided a lot of personal and skill expression.
